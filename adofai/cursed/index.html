Curse Mechanism

In the following content, midspins will be considered as 0° angle.

Basics

Curse is a type of object that can be applied on ADOFAI Levels which performs the corrsponding transform action for parts that meets certain conditions.
A curse is usually binded with a certain ADOFAI level, and use the (slightly modified) ID of the level as the ID of the curse.
Curse effects that don't binds to levels also exists, for example, Color Curse Series by 小绿君.

But Statement

A brief introduction to describe a curse effect is called a but statement.
But statements are usually written after the title of the level that the curse effect belongs to, for example:
1-X? A Dance of Fire and Ice but the music is cursed
2-X? Offbeats but it's a little longer
This website recommends using the following level title format:
{level ID} {level title} but {curse effect introduction}
The actual effect might be different between what the but statement describes.
The time order of curse effects IS NOT the affect order of curse effects, and there's no direct relationship between them.

Formats

The curses the player would experience, including the currents and the nexts, could be tipped by using text decorations on the bottom of the interface.
Current curse: "12-X"
Next curse: "XT-X"
In addition, if the curse effects are all from 1 single series:
Current curse by Yangsy56302: 12-X
Next curse by Yangsy56302: XT-X
The tips shouldn't move or rotate with the camera, or they will not have a prompt effect. although it's already not understandable for most of the viewers...
There will be a countdown before updating the current curse effects:
The countdown starts with 3, decreases by 1 in equal interval time. The curse effects will be updated at the exact same time when the number (should be) counted down to -1.
The countdown usually is located at the end of the line before the colon, surrounded by brackets, with a extra space before the left bracket only:
Current curse: "ML-X"
Next curse (3): "5-X"

Current curse: "ML-X"
Next curse (2): "5-X"

Current curse: "ML-X"
Next curse (1): "5-X"

Current curse: "ML-X"
Next curse (0): "5-X"

Current curse: "5-X"
Next curse: "2-X"
Multiplication signs * are used to present that multiple curses takes effect in order from left to right.
Function composition signs ∘ are used to present that multiple curses takes effect in order from right to left.
Here's an example:
Current curse: "XR-X" * "12-X"
Next curse: "XR-X" ∘ "12-X"

"XR-X" * "12-X" will first add triangles,
then try to change the angle of every tile,
including those tiles that is a part of these new triangle,
so now these triangles are probably not triangles anymore;

but "XR-X" ∘ "12-X" will first try to change the angle of every tile,
then add triangles which will not be modified by the angle change process thats already finished.
Nesting curse functions is another way to specify the curse order:
The innermost function will take effect first, and the leftmost function will take effect last.
Current level: "12-X"( "XR-X"( XC-X ) )
Next level: "XR-X"( "12-X"( XC-X ) )
However, abusing nesting would cause a lot of brackets to stack, therefore the hints would be even harder to parse.
The notation method that uses plus sign + to represent that multiple curses takes effect in order from left to right has been DEPRECATED!

Keywords

None
Special curse effect that doesn't do anything actually.
Current curse: None
...
Usually lonely appears at Next curse to represent please wait.
Next curse: ...
Sometimes could alsos be used as need more info: If some part of the curse hint wouldn't change in a while, while the other part changes frequently, Ellipsis could be used to represent that "other" part which could be noted at somewhere else.
Current curse: "12-X" * "12-X"
Next curse: "12-X" * ... * "12-X"
StopIteration
The curse effect won't change before level ends, so the player can now finally ignore the hint and fully focuses on the chart.
Next curse: StopIteration

Meta Mechanism

If there's need to describe that if multiple curses with same ID are the same instance or not, please use subscripts:
Current curse: "3-X"₁
Next curse: "3-X"₂
Current curse: "3-X"_1
Next curse: "3-X"_2
(Thanks Regularly for the format method!)
Repeated curse effects can be represented by the power operation of functions:
Current curse: "XS-X" * "XS-X" * "XS-X"
Next curse: "XS-X"3
or
Next curse: "XS-X" ^ 3
Reverted Curses
Reverted (recommended) / Inverted / Negated / Anti- Curses is basically a method to "revert" the effect of a curse.
An extra condition is that part is possible to have an reverted version which is possible to form the current part if you use the original curse to it;
Otherwise, you can still use them, but it won't do anything when the preconditions no longer mets, until it mets again.
Reverted curses are represented by the power operation of functions too:
Current curse: "XS-X"-1 * "XS-X"-1 * "XS-X"-1
Next curse: "XS-X"-3
or
Next curse: "XS-X" ^ -3
In the deprecated sequential format,
Reverted curses were represented by changing the - sign to the + sign in the ID of the corresponding curse, or by inserting the - sign at the beginning of the ID.
Current curse: "8+X"
Next curse: "9-X" - "9-X"
Sequential format allowed to abbreviate 2 opposite curses which are next to each other.
Current curse: "XC-X" - "XC-X"
Next curse: ± "XC-X"
Current curse: - "XC-X" + "XC-X"
Next curse: ∓ "XC-X"
(Thanks DragonFire28 for the suggestion of the color! )
Using a reverted curse may produces many possibilities of results which is totally different with the original section, or does not match expectations at all.
