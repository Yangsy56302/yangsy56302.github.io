Concept definitions for ADOFAI objects

This is not (and almost impossible to be) a set of standard definitions that everyone must follow.
This is just one set of definitions to attempt to disambiguate when communicating between different people using different definitions.

Tile

The concept of tile is trivial, but the details are not.

Each tile has a tile number starts from 0 (so the "first" tile is technically the zeroth tile).

Direction

The concept of direction is also trivial.
Also just in case you don't know, 0° is right, and 90° is up.

Absolute Direction

The (absolute) tile direction of a tile is the same as the direction from the selected tile to the button that to create this tile in editor,
or the same as the direction of an arrow, starts at the center point of the previous tile, and ends at the center point of itself.

ADOFAI stores tile angle data using absolute direction in degrees (exceptions are midspins, since their absolute direction are the same with turnarounds, they use the hardcoded value of 999).

Relative Direction

The relative tile direction of a tile is the same as the absolute tile direction, except that selected | previous tile is now considered as 0°.

Angle

The angle of a tile depends on:

  • whenever Twirls are considered (true by default)
  • whenever Multi Planets are considered (false by default)
  • whenever Paused Beats are considered
  • whenever extra hold loops are considered

Original Angle

The original angle would ignore all of above.

Turnaround's original angle is 360°, and midspin's original angle is 0°.

Directional Angle

The directional angle would only consider Twirls:

it's the same as the original angle by default,
and 360° - {OriginalAngle} if the planet is rotating counter-clockwise and the tile isn't turnarounds nor midspins.

Extended Angle

The extended angle would also consider Paused Beats and extra hold loops:

{ExtendedAngle} = {DirectionalAngle} + 180° * {PausedBeat.Duration} + 360° * {Hold.ExtraHoldLoops}

Gameplay Angle

The gameplay angle would also consider Multi Planets:

{GameplayAngle} = {ExtendedAngle} - {MultiPlanetAngle} , where
{MultiPlanetAngle} = ({NumberOfPlanets} - 2) * 180° / {NumberOfPlanets}
